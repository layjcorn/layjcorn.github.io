SENSOR-ACTUATED VALVE SYSTEM

The core capability of this system is closed-loop jam detection. The primary function detects hand presence via an infrared QRD sensor, triggering a Parallax Propeller 2 microcontroller to drive a PZ22 DC gear motor through an inverted-logic relay module.

Rather than relying on simple software timeouts, the system autonomously detects and recovers from mechanical valve jams by measuring motor current draw across a 5Ω shunt resistor passed through an LM358N op-amp. This moves the project from a basic open-loop actuator into a robust controls engineering implementation.

pub HandSense()
  repeat
    hand := pinread(0)
    debug(`handstatus '`(hand)' 10)
    waitus(500)

' Inside Main() Loop:

' Start Cog 1 running Encoder method
cogspin(1,Encoder(),@Stack1)                             

' Inside Main() Loop:
pinlow(4)                                         ' IN1 on (black) - starts CCW rotation
repeat until position/4 == 66                     ' keep IN4 on until 495deg (24T/10T * 495deg/360deg * 20/turn = 66)
pinhigh(4)                                        ' IN1 off (black) - stops CCW rotation
wait := 5000                                      ' Initialize 'wait' variable @ 5000

' ...after valve opens:
waitms(100)
repeat until hand == 0 or wait <= 0               ' loops until hand is removed OR 5 seconds pass
  waitms(1)                                       ' waits one second
  wait--                                          ' decreases variable incrementally until time's up
pinlow(5)                                         ' IN2 on (red) - starts CW rotation
repeat until position == 0                        ' repeats until in home position (assuming initialized at 3 o'clock)
pub Jammer()
  repeat
    if ina.[4] == 0 or ina.[5] == 0               ' only sense when a relay is on (otherwise it will read 0V)
      jam := pinread(1)
    else                                          ' set to 1 when motor is stopped
      jam := 1
      waitms(100)

' Inside Main() Loop:
if jam == 0                                       ' if pin 1 reads 0 = shunt resistor detected jam
  waitms(500)                                     ' quick wait and double-check (was getting too many false positives)
  if jam == 0                                     ' double-check it's a real jam
    numjams++                                     ' keep track of # of jams
if numjams == 1                                   ' if first jam open up and try to close again
  pinhigh(5)                                      ' IN2 off (red) - stops CW rotation
  waitms(100)
  pinlow(4)                                       ' IN1 on (black) - starts CCW rotation
  repeat until position/4 == 66                   ' go back to 495deg
  pinhigh(4)                                      ' IN1 off (black) - stops rotation
  waitms(100)
  pinlow(5)                                       ' IN2 on (red) - starts CW rotation
  waitms(100)
pinhigh(5)                                        ' IN2 off (red) - stops CW rotation
repeat until hand == 0                            ' ensure hand is removed before next iteration
waitms(100)
if numjams == 2                                   ' second jam stops everything
  pinhigh(4)                                      ' stop both relays
  pinhigh(5)                                      ' power off! (both ways)
  repeat                                          ' will cycle forever until entire board powered off
    pintoggle(56)                                 ' Toggle P56 (initially from on to off)
    pintoggle(57)                                 ' Toggle P57 (initially from off to on)
    waitms(100)

CAM PROFILE ACCURACY

This project isolates a specific engineering problem: which motor type more accurately reproduces a defined, non-linear cam profile, and by how much? By treating the motor type as the independent variable and profile tracking accuracy as the dependent variable, we evaluated both platforms under identical proportional control structures. The stepper motor ultimately produced significantly higher accuracy due to its fundamental step-resolution mechanics and dynamic microstepping capabilities.

KINEMATIC MATH & MICROSTEPPING

To accurately reproduce the piecewise waveform, target velocity (v) was calculated in real-time across four distinct mathematical bounds using Propeller 2 parallel processing. Base speed was dynamically converted into a microstepping WaitTime.

pub Math() | y, v, base_speed, temp_val
  repeat
    y := float(Position) /. 3200.0
    if direction == 1
      if Position < 1440
        v := 3.14159 *. fsqrt(0.2025 -. (y *. y))
      elseif Position < 1760
        temp := 1.0 -. ((y -. 0.45) /. 0.05)
        v := 0.47124 *. fsqrt(1.0 -. (temp *. temp))
      else
        temp := 1.0 -. ((y -. 0.55) /. 0.45)
        v := 1.4137 *. fsqrt(1.0 -. (temp *. temp))
    else
      temp := (2.0 *. y) -. 1.0
      v := 2.35619 *. fsqrt(1.0 -. (temp *. temp))
    WaitTime := trunc(1500 /. (max(v, 0.1)))

TEST METHODOLOGY & RESULTS

This project evaluated motor type (Stepper vs. Servo) as the independent variable against profile tracking accuracy. The stepper motor produced 93% lower RMS error.

Result: Stepper motor microstepping provided superior non-linear tracking resolution.

REAL-TIME MULTIPLAYER INPUT

The Propeller 2 is a multi-core microcontroller featuring eight independent parallel processors. Moving beyond standard single-core embedded applications, this project implemented a "Whack-a-Mole" style game by distributing concurrent hardware input handling, game state logic, and VGA display rendering across separate physical cores. This architecture avoids standard interrupt-driven timing conflicts and introduces complex considerations for inter-core communication and shared memory access.

Work was divided efficiently: Core 0 was dedicated exclusively to generating the VGA timing signals, Core 1 maintained the authoritative game state and scoring, and Core 2 acted as an autonomous input driver, reading HID data over I2C and dropping sanitized player coordinates into a shared memory register.
